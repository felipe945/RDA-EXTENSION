# POOL — Terminal 2: EXTENSION (per-rep personal touches, FanBasis-only marking)

## MISSION
In the extension: (1) **only FanBasis-account sends set stage "DM Sent"** — a personal-IG send records a per-rep touch and nothing else (no stage, no due_at, lead stays queued); (2) the Personal chip shows **the signed-in rep's own state**, not the team's; (3) all touch writes go through the new server endpoint that stamps rep identity (Contract TOUCH). Felipe's locked model: "personal-first stays queued".

## CONTRACT TOUCH (provided by T1)
`POST {dashboardUrl}/api/leads/touch`, bearer repToken, body `{ leadId, channel: "ig_fanbasis" | "ig_personal", sent?: boolean }` → `{ ok, outreach_channels }`. The server stamps WHO from the token — never send rep identity from the client. Response `outreach_channels` contains the updated shape:
```
{ ig_fanbasis: {sent, sentAt, byId, byName},
  ig_personal_by: { [repId]: {sent, sentAt, name, handle} },
  ig_personal: {sent, sentAt} }   // legacy aggregate — DO NOT write it anymore; read only as fallback
```
Rep identity for READS: `boot.rep.id` from the cached bootstrap (`fb_bootstrap`, shape `rep: {id, email, name, personalIgUsername}` — background.js caches it after sign-in).

case "TOUCH_LEAD": {
  try {

      method: "POST",
      headers: { "Content-Type": "application/json", ...bearer },
      body: JSON.stringify({ leadId: msg.id, channel: msg.channel, sent: msg.sent !== false }),

    if (!res.ok) throw new Error(String(res.status));

    // Refresh the lead cache so chips re-read the new outreach_channels

  }
  break;
}
